# 3. 未完成工作总表（按优先级）

## 3.1 P0（必须完成，影响主链验收）

### A. Workbench 输入绑定动作未闭环

- 现状：Inputs 区主要为结构展示，缺少真实上传/绑定/替换流程。
- 缺口：
  - Upload Asset
  - Bind / Replace / Remove
  - Invalid Binding 高亮与修复路径
- 对应验收：`09` 文档 `3.3.2`、`3.3.6`

### B. Tasks 页面筛选能力不足

- 现状：有列表，但缺完整筛选入口。
- 缺口：
  - 搜索框
  - 状态 Tabs（waiting / running / failed / succeeded）
- 对应验收：`09` 文档 `3.4.1`、`3.4.2`

### C. Governance 页结构尚不完整

- 现状：有 Summary / Events / Artifacts / Audit 摘要。
- 缺口：
  - Manifest Tab
  - Governance Panel（required actions / suggested fix 独立主区）
  - 失败恢复与取消场景的更明确动作区
- 对应验收：`09` 文档 `3.5.1`、`3.5.3`、`3.5.4`

### D. Results 与 Result 消费链路未完全打通

- 现状：有 Results 列表与 Result Detail。
- 缺口：
  - Results 列表页（按 Scene / Analysis Type / Model / Time 筛选）
  - Result Detail（Map + Metrics + Explanation + Export + Context）
- 对应验收：`09` 文档 `3.7`

# 7. 交付定义（“完成”标准）

每一项工作完成时，至少满足：

- 有对应页面或组件落地
- 有状态行为（非仅静态 UI）
- 有跨区块联动（非孤立展示）
- 可被 `09-前端页面验收清单` 对应条目勾选通过
