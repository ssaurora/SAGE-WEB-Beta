| `AssetDetailViewModel` | 资产详情页 |

---

## 3. DTO / Mock / API 约定

- `lib/contracts/task.ts` 和 `lib/contracts/asset.ts` 负责定义统一类型。
- `lib/mock/task.ts` 和 `lib/mock/asset.ts` 负责提供 mock 数据，并可作为本地联调数据源。
- `lib/api/task.ts` 和 `lib/api/asset.ts` 负责对外暴露页面可消费的 ViewModel。
- 页面组件只消费 ViewModel，不直接依赖 mock 文件中的类型定义。

## 5. 后续联调建议

如果后端开始提供真实 Task 或 Asset 数据：

1. 先替换 `lib/api/task.ts` 或 `lib/api/asset.ts`
2. 再检查返回结构是否与对应 contracts 一致
3. 最后仅调整适配层，避免页面层联动修改
