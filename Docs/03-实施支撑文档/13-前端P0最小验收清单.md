# 前端 P0 最小验收清单

**版本**：v1.0  
**日期**：2026-04-03  
**范围**：P0-A ~ P0-D（Workbench / Tasks / Governance / Reports）

---

## 1. Workbench（P0-A）

- [x] 进入 Workbench 页面后，可看到 Inputs 交互区与 Layers 区。
- [x] 点击 Upload 可新增未绑定资源。
- [x] Required Input 支持 Bind / Replace / Remove。
- [x] Optional Input 支持 Bind / Replace / Remove。
- [x] 绑定后 Required Ready 计数变化正确。
- [x] 只读状态下输入动作按钮不可用。

## 2. Tasks（P0-B）

- [x] Tasks 页面存在搜索框。
- [x] 支持按状态筛选（All / Waiting / Running / Failed / Succeeded）。
- [x] 搜索与状态筛选可叠加生效。
- [x] 显示“当前显示 X / Y”计数。
- [x] 点击任务可进入对应治理页。

## 3. Governance（P0-C）

- [x] Governance 页面显示状态摘要（可恢复/可取消/结果可用）。
- [x] 失败可恢复时显示 Fix and Resume 路径。
- [x] 失败不可恢复时显示 Start New Analysis 路径。
- [x] Cancelled 状态显示 Re-run 路径。
- [x] Waiting/Running 状态不误判为 Cancelled。
- [x] Manifest 区可展示参数树（输入/参数/输出）。

## 4. Reports（P0-D）

- [x] Reports 列表页可加载并展示报告卡片。
- [x] 列表页支持搜索、格式筛选、状态筛选、排序。
- [x] 点击报告可进入详情页。
- [x] 详情页展示元信息、sections、metadata、回溯入口。
- [x] Reports 数据来源走 `/api/reports`（mock API），非页面硬编码。

## 5. 构建与回归

- [x] 执行 `npm run build` 通过。
- [x] 关键路由可访问：`/tasks`、`/task-governance/[taskId]`、`/reports`、`/reports/[reportId]`。
- [x] 无新增 TypeScript 类型错误。
